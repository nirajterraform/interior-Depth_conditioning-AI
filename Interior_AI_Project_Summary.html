3. Approaches Tried

Abandoned

Approach 1 — Multi-image FLUX Pass 1

Send the room photo + all product images simultaneously to FLUX. Prompt references each product as @image2, @image3, etc.

Result: Geometry dropped to 52–68%. FLUX tried to reconcile all visual references at once and reinterpreted the room structure.

Why abandoned: Geometry preservation is more critical than product fidelity. Room architecture breaking kills the end product experience.

Current — Pass 1

Approach 2 — Text-only FLUX Pass 1

Gemini Vision writes a 1–2 sentence visual description of each product (colour, material, silhouette, style). FLUX receives only the room photo + text descriptions — no product images in Pass 1.

Result: Geometry 80–88%, catalogue 70–90% style match.

Trade-off: Products in the output are AI-generated furniture inspired by the catalogue items — not pixel-exact product images.

Abandoned

Approach 3 — 3-product Targeted Edit Loop (Pass 2)

After Pass 1, run 3 sequential targeted swaps — one per lowest-scoring product — using the actual product photo as FLUX reference. Adopt if the total catalogue average improves.

Problem: Gemini scoring noise on the 2 unedited products dragged the catalogue average down even when the edited product genuinely improved. Result was almost always discarded.

Why abandoned: 3 edits wasted ~3 min and were consistently rejected due to the broad adoption gate.

Current — Pass 2

Approach 4 — Single Targeted Edit (Pass 2)

After Pass 1, if catalogue avg < 65% AND any product scored < 60%, run exactly 1 targeted swap for the worst-scoring product using the actual product photo as FLUX reference. Adopt only if that specific product's score improves.

Geometry gate: Targeted edit is discarded if geometry drops more than 8 pts from Pass 1.

Still needs validation across multiple test runs — not yet confirmed reliable.
Not Built

Approach 5 — True Compositing

Segment furniture zones → remove background from each product image → warp to match room perspective and scale → AI-match lighting and shadows → composite into scene.

WOW factor: Highest — customer sees their exact product in their actual room.

Why not built: Requires segmentation model + depth estimation + shadow matching AI. Completely different pipeline architecture.

5. Current Architecture Flow

1

Room type detection

Gemini Vision analyses the uploaded room photo and returns roomType (living_room, bedroom, etc.) and theme.

2

Catalogue retrieval

Supabase vector search retrieves products matching the room + theme. Products are scored, filtered by room-wrong-terms penalty, deduplicated by normalized_category (1 per category), and up to 6 are forwarded.

3

Product visual descriptions (Gemini)

For each product, Gemini Vision writes a 1–2 sentence description covering material, dominant colour, silhouette, texture, and style cues. Used as text input to FLUX instead of product images.

4

Pass 1 — Generation + validation (up to 3 attempts)

  • FLUX receives: room photo + text descriptions only (no product images)
  • Geometry validated by rembg (pixel diff) + Gemini (structural scoring)
  • Catalogue scored by Gemini per product — averaged over requested products only
  • Accept if: geometry ≥ 72% AND gemini_geometry ≥ 55% AND catalogue ≥ 50%
  • Each retry adds escalating correction directives to the prompt
5

Pass 2 — Single targeted edit (conditional)

Triggers only if: catalogue avg < 65% AND at least one product scored < 60%.

  • Picks the single worst-scoring product
  • Uploads actual product photo to fal storage as visual reference
  • FLUX swaps that item using the product image
  • Adopted if: that product's score improves AND geometry drop ≤ 8 pts AND no hallucination
6

Return result

Returns best accepted result with validation scores, placed products, invented item bounding boxes, and scale issues. If all 3 attempts fail → returns best effort result with BEST EFFORT badge.

65%

8 pts

Geometry merged score = Gemini × 0.35 + rembg × 0.65. rembg is weighted higher as objective pixel measurement. MIN_GEMINI_GEOMETRY prevents rembg from inflating the merged score when Gemini detects a clear structural failure (e.g. floor material change carpet→tile, wall repaint, camera shift).

9. Known Limitations

No pixel-exact product placement

FLUX interprets product images as style references and generates new pixels. It cannot paste a product photo into a scene. True compositing requires a completely different architecture.

Gemini geometry false positives

Gemini occasionally scores geometry at 80–100 even when camera angle subtly shifts. MIN_GEMINI_GEOMETRY catches major structural failures but subtle perspective drift can still pass.

Hallucination on sparse product sets

When only 2 products pass dedup, FLUX fills empty space with hallucinated furniture. The hallucination flag is logged but NOT in the acceptance gate — only geometry + catalogue scores block acceptance.

Kids products surfacing in bedroom

"Toddler Bed Tent Canopy Night Light" and similar products surface in bedroom searches and are miscategorised as "lamp". This is a catalogue data quality issue, not a code issue.
