2. Problem Statements

P1 — Geometry Drift CRITICAL

What happens: FLUX changes architectural elements it should never touch — floors, fireplaces, archways, wall layout, curtains.

Examples seen: Tile floor → wood (TC3), fireplace replaced (TC2), arch doorway removed (TC3), walls repainted (TC1).

Root cause:

  • Fireplace, archways, and floor material are NOT explicitly listed in the geometry lock prompt — FLUX treats them as decorative
  • The objective pixel-level geometry check (rembg) was removed during speed optimisation
  • Gemini-only geometry scoring is subjective and too lenient
P2 — Validation Scores Are Unreliable HIGH

What happens: Gemini reports 95% geometry and 95% catalogue match when the output is visually clearly wrong.

Examples seen: Fireplace completely changed but geometry scored 95% (TC2). Sofa visually different from catalogue but scored 95% match (TC2).

Root cause:

  • Gemini evaluates holistically — sees walls/ceiling/windows are same, gives high score without checking fireplace or floor individually
  • Catalogue matching checks "is a sofa present?" not "does this sofa look like the reference sofa?"
  • Users receive false confidence from scores
P3 — Scale Errors (Furniture Proportions Wrong) MEDIUM

What happens: Furniture rendered at incorrect scale relative to other pieces and the room.

Examples seen: Bench rendered same height/size as bed (TC4). Lamp rendered larger than sofa (earlier tests).

Root cause:

  • Generation prompt lists all products equally with no size hierarchy
  • FLUX has no understanding of real-world furniture proportions
  • Nothing establishes that a bed is larger than a bench
P4 — Targeted Edit Placement (Floating / Hanging Items) MEDIUM

What happens: When a user adds a product via text edit or "Try in Room", the item appears floating or without physical support.

Examples seen: Side lamp mounted on nothing, appearing to hang in mid-air (TC3).

Root cause:

  • Targeted edit prompt gives no grounding or placement instruction
  • FLUX places items visually without understanding gravity or physical support
P5 — Catalogue Visual Match Is Category-Based Not Visual HIGH

What happens: Validation says catalogue product matched but the generated item looks completely different from the selected product.

Examples seen: Specific sofa selected — completely different sofa generated — validation scored 95% (TC2).

Root cause:

  • Gemini validation asks "is this product present?" — answered by category match only
  • Does not compare visual appearance (colour, shape, material, silhouette) against the reference product image
P6 — AI Invented Items Panel — Poor Image Quality LOW

What happens: Crops shown in the AI Invented Items panel are dark, blurry, tiny, or unrecognisable.

Root cause: Crop bounding boxes from Gemini sometimes point to dark corners. No brightness filter was applied. Minimum crop size was too small.

Status: Partially resolved — brightness threshold and 80px minimum added.

P7 — Image Quality Degradation in Dark Rooms MEDIUM

What happens: Generated image is grainy and lower quality than the original photo when source is dark/low-light.

Root cause: FLUX over-processes dark images to "fix" lighting, introducing noise. This is a fundamental model limitation — not a prompt issue.

Status: No prompt fix available. Would require pre-brightening the source image.

3. Proposed Fixes

Fix 1 — Strengthen Geometry Lock Prompt

What to change: Add the following to the locked elements list in buildStrictEditPrompt and buildGeometryLock:

  • Fireplace: surround, style, material, presence — must not be changed or replaced
  • Archways and room entry frames: structural openings must remain exactly as-is
  • Floor material and colour: tile stays tile, wood stays wood, carpet stays carpet
  • All ceiling fixtures: fan blades, chandelier, pendant lights — unchanged
  • Accent wall panels and decorative wall treatments

Files affected: app/api/fal-place-products/route.ts, app/api/targeted-edit/route.ts

Time impact: Zero — prompt only
Cost impact: Zero
Resolves: P1 (60% confidence)
Fix 2 — Restore rembg Pixel-Level Geometry Score

What to change: Re-enable buildBackgroundMaskScore which was removed during speed optimisation. Uses the rembg model to isolate non-furniture pixels and compare them between original and generated image mathematically.

Why it matters: This is the ONLY objective, non-AI measurement of geometry change. It catches floor changes, wall repainting, and fireplace changes that Gemini misses.

Files affected: app/api/fal-place-products/route.ts

Time impact: +15s per generation (78s → ~93s)
Cost impact: ~Zero (rembg is a fal.ai utility)
Resolves: P1 (85%), P2 partial (70%)
Fix 3 — Add Scale Hierarchy to Generation Prompt

What to change: Add a scale rules section to buildStrictEditPrompt:

SCALE HIERARCHY — real-world proportions must be respected:
• Hero pieces (sofa, bed, dining table): largest items, 70-90cm tall
• Secondary pieces (chair, side table, lamp): medium, 45-75cm tall
• Accent pieces (bench, ottoman, stool): smallest seating, 40-50cm tall
• A bench must NEVER appear larger than a bed or sofa
• A lamp shade must NEVER appear wider than a sofa back
• Use door frame (standard 200cm tall) as the room scale anchor
• If in doubt, make the item smaller rather than larger

Files affected: app/api/fal-place-products/route.ts

Time impact: Zero — prompt only
Cost impact: Zero
Resolves: P3 (70% confidence)
Fix 4 — Stricter Gemini Validation Prompt

What to change: Update the validation prompt in validateWithGemini:

  • Geometry: Explicitly ask Gemini to check fireplace, floor material, archways, curtains individually — not just walls/windows/ceiling
  • Catalogue match: Ask "does the generated item visually match the reference — same dominant colour, material, silhouette?" not just "is the item present?"
  • Proportionality: Add check "are all items at realistic real-world scale relative to each other?"

Files affected: app/api/fal-place-products/route.ts

Time impact: Zero — same Gemini call
Cost impact: Zero
Resolves: P2 (80%), P5 (70%), P3 partial (50%)
Fix 5 — Restore MAX_ATTEMPTS to 2

What to change: Change MAX_ATTEMPTS from 1 back to 2. If attempt 1 fails geometry or catalogue check, attempt 2 receives a retry directive telling FLUX exactly what failed.

Why it matters: In testing before optimisation, the retry directive "geometry drift detected — preserve floor, walls, fireplace exactly" significantly improved attempt 2 quality.

Files affected: app/api/fal-place-products/route.ts

Time impact: +0s if attempt 1 passes. +78s worst case
Cost impact: +30-40% on failed generations only
Resolves: P1 (65% confidence)
Fix 6 — Targeted Edit Grounding Instruction

What to change: Add a physical grounding rule to both buildTextEditPrompt and buildProductSwapPrompt:

PHYSICAL GROUNDING — all items must obey gravity:
• Floor lamps: base must rest visibly on the floor
• Table lamps: base must rest on a table or shelf surface
• Wall art: must be mounted flat against a wall
• Sofas, chairs, benches: legs or base must contact the floor
• No item may appear floating, hovering, or hanging without visible support

Files affected: app/api/targeted-edit/route.ts

Time impact: Zero — prompt only
Cost impact: Zero
Resolves: P4 (80% confidence)
